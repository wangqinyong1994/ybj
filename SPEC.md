# 居家服务网站 SPEC 文档

## Problem Statement（问题陈述）

公司提供新房开荒、深度保洁、甲醛治理三项居家服务，目前缺乏线上获客渠道。需要建设一个专业、可信的服务展示网站，吸引潜在客户，引导他们通过电话或微信咨询下单，实现线上获客到线下成交的转化。

## Proposed Solution（方案描述）

### 核心定位
**线上获客型展示网站** - 不涉及在线交易，核心目标是展示服务、建立信任、引流到私域（电话/微信）完成成交。

### 页面结构
| 页面 | 说明 |
|------|------|
| 首页 | 服务概览、核心优势、快速预约入口 |
| 新房开荒 | 服务介绍、流程、范围、案例、价格说明、预约表单 |
| 深度保洁 | 服务介绍、流程、范围、案例、价格说明、预约表单 |
| 甲醛治理 | 服务介绍、流程、范围、案例、价格说明、预约表单 |
| 服务案例 | 按服务分类 + 时间线展示 + 综合筛选，前后对比图 |
| 保洁知识/资讯 | 行业知识、保洁技巧，用于SEO引流 |
| 关于我们 | 公司介绍、资质证书、服务承诺 |
| 联系我们 | 电话、微信二维码、地址、预约表单 |

### 服务详情页模块（每项服务）
- 服务介绍
- 服务流程
- 服务范围
- 服务承诺/保障
- 用户评价（来源标注）
- 服务视频
- 案例展示
- 价格说明（询价模式，不展示具体价格）
- 在线预约表单
- 常见问题

### 核心功能
1. **在线预约表单** - 用户提交后信息发送到指定邮箱/微信，人工跟进
2. **联系方式展示** - 微信二维码 + 电话号码双通道并列
3. **案例筛选** - 按服务类型、面积、地区等筛选
4. **SEO优化** - 深度SEO + 百度统计/Google Analytics 数据追踪

### 设计风格
- **主色调**：rgb(28,118,44)（绿色）- 环保、健康、专业
- **风格定位**：专业可信 + 温馨亲和互补
- **响应式**：PC端和移动端同等重要，精心设计

## Technical Constraints（技术约束）

| 约束项 | 说明 |
|--------|------|
| 技术栈 | SSR框架（Nuxt.js 或 Next.js），SEO友好 |
| 无用户系统 | 无需登录注册，预约无需账户 |
| 无后台管理 | 内容由技术人员直接维护更新 |
| 单语言 | 仅中文，服务本地市场 |
| 单城市 | 无需城市选择或地区切换 |
| 部署要求 | 支持SSR部署（Node.js环境或静态生成） |
| 兼容性 | 支持主流浏览器（Chrome、Safari、Firefox、Edge）及移动端 |

## Non-goals（明确不做的事）

1. ❌ 在线支付功能 - 不支持网上下单支付
2. ❌ 用户注册登录系统 - 无需用户账户体系
3. ❌ 后台管理系统 - 内容由技术人员维护
4. ❌ 在线客服聊天窗口 - 仅展示联系方式
5. ❌ 具体价格展示 - 采用询价模式，引导咨询
6. ❌ 多语言支持 - 仅中文
7. ❌ 多城市/地区切换 - 单城市服务
8. ❌ 第三方平台对接（如美团、大众点评）- 独立网站

## Success Criteria（成功标准）

### 上线标准
- [ ] 1-2周内完成开发并上线
- [ ] 所有页面在PC端和移动端正常显示
- [ ] 所有预约表单可正常提交并收到通知
- [ ] SEO基础配置完成（标题、描述、关键词、sitemap）
- [ ] 统计代码接入完成（百度统计/Google Analytics）

### 业务标准
- [ ] 网站上线后，用户能清晰了解三项服务的区别和流程
- [ ] 预约表单提交后，运营人员能及时收到通知并跟进
- [ ] 用户能在3次点击内找到联系方式或预约入口
- [ ] 网站在搜索引擎可被正常收录

## 附录：需求访谈记录

### Q1: 目标用户
- **选择**: 线上获客为主 - 重点是展示服务、吸引潜在客户，让他们通过电话或微信咨询下单

### Q2: 服务类型
- **选择**: 三种都做 - 新房开荒、深度保洁、甲醛治理各一个服务页面

### Q3: 服务展示页内容模块
- **选择**: 丰富版本 - 服务介绍、服务流程、服务范围、服务承诺/保障、用户评价、服务视频、在线预约表单、常见问题

### Q4: 预约表单处理
- **选择**: 纯通知 - 表单提交后，信息发到指定邮箱/微信，人工跟进

### Q5: 用户系统
- **选择**: 不需要 - 纯展示型网站，用户无需注册，预约表单无需登录

### Q6: 后台管理系统
- **选择**: 不需要 - 内容由技术人员直接修改代码/配置文件更新

### Q7: 访问渠道
- **选择**: 双端同等重要 - PC端和移动端都需要精心设计，响应式布局

### Q8: 页面结构
- **选择**: 完整配置 - 首页、关于我们、联系我们、三个服务页、服务案例页、保洁知识/资讯板块

### Q9: 服务案例展示方式
- **选择**: 都要 - 按服务分类 + 时间线展示 + 综合筛选功能

### Q10: 资讯内容来源
- **选择**: 纯手工发布 - 所有文章由内部人员撰写，技术人员更新到网站

### Q11: 多语言支持
- **选择**: 单语言 - 只需要中文，服务本地市场

### Q12: SEO优化
- **选择**: SEO + 数据追踪 - 深度SEO优化 + 接入统计工具追踪流量

### Q13: 价格展示
- **选择**: 询价模式 - 不展示具体价格，引导用户咨询后根据实际情况报价

### Q14: 品牌形象/设计风格
- **选择**: rgb(28,118,44)主色调，专业可信 + 温馨亲和互补

### Q15: 联系方式优先级
- **选择**: 双通道并列 - 微信和电话同等重要，根据页面位置灵活展示

### Q16: 在线客服
- **选择**: 不需要 - 只展示微信和电话，不做在线聊天窗口

### Q17: 服务地理范围
- **选择**: 单城市 - 只服务一个城市，网站内容不需要体现地区

### Q18: 用户评价真实性呈现
- **选择**: 来源标注 - 评价标注来源，增加可信度

### Q19: 上线时间
- **选择**: 尽快上线 - 1-2周内上线，后续按需小幅调整

### Q20: 技术选型
- **选择**: 服务端渲染 - 使用SSR技术（如Nuxt/Next.js），对SEO更友好
